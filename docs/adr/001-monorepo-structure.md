# ADR 001: Структура Монорепозитория

## Статус

Принято (Accepted)

## Контекст

Нам необходимо управлять тремя различными сервисами:

1. Фронтенд (Next.js)
2. Основной бэкенд (NestJS)
3. Движок телеметрии (Go)

Нам нужно обеспечить удобство совместной разработки, общие типы и простоту развертывания.

## Решение

Мы выбрали архитектуру **Monorepo** с использованием **npm workspaces**.
Все приложения находятся в папке `apps/`.
Общая документация и ADR — в папке `docs/`.

## Последствия

- (+) Легко управлять зависимостями из одного корня.
- (+) Возможность делиться типами между фронтендом и бэкендом.
- (+) Единый CI/CD пайплайн.
- (-) Размер репозитория будет расти быстрее.
- (-) Требуется настройка инструментов (линтеры, тесты) для работы с воркспейсами.
